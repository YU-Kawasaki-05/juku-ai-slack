# FR-05: AI回答生成（適応型応答戦略）

## 概要

生徒の質問に対して、P(mastery) 値に基づく3モード適応型応答戦略（DEC-23）で回答を生成する。
Tutor LLM（回答・確認質問生成）と Evaluator LLM（正誤シグナル抽出）を分離し（DEC-24）、
日本の中高生向け「伴走者」フレーミング（DEC-25）を全回答で徹底する。

## アーキテクチャ図

```
生徒メッセージ
    │
    ├─ [1] P(mastery) ルックアップ (FR-23) + トピック検出
    │       └→ モード決定 (direct / socratic / confirmation)
    │
    ├─ [2] Tutor LLM — 回答 + 確認質問を生成
    │       └→ Slack に返信
    │
    └─ [3] Evaluator LLM — waitUntil()で非同期実行 (FR-23)
            └→ BKT 更新
```

## アクター

- U-01 生徒（質問者）
- U-02 スタッフ（質問者、スタッフ向け補足を要求する場合もあり）

## 入力データ

| フィールド | 型 | 必須 | 制約 | 備考 |
|-----------|----|----|------|------|
| 質問テキスト | string | ○ | - | メンション部分を除去したもの |
| 生徒プロフィール要約 | string | ○ | 500〜1500 tokens程度 | FR-09から取得 |
| 関連RAGチャンク | string[] | △ | 上位3〜8件 | FR-10から取得 |
| スレッド履歴 | message[] | △ | 直近数往復 | FR-03セッションから取得 |
| 添付画像 | base64 or URL | △ | jpg/png/webp、1〜3枚 | FR-06から取得 |
| P(mastery) 知識状態 | object | △ | topic別 float0〜1 | FR-23から取得 |
| 試験前モードフラグ | boolean | △ | - | student_profiles.exam_mode_until から |
| スタッフ指示フラグ | boolean | △ | - | 「スタッフ向けに」等の指示を含む場合 |

## 出力 / 結果

- 正常系: Slackの元メッセージへのスレッド返信（thread_ts付き）
- 異常系: エラーコードに対応するユーザー向け文言をSlackに返信

## 3モード適応型応答戦略（DEC-23）

P(mastery) の値と試験前モードに応じてモードを決定する。

| 条件 | モード | 応答スタイル |
|------|--------|------------|
| 試験前モード ON | **direct** | 要点・スピード優先、確認質問なし |
| P < 0.3 | **direct** | 直接指導 + ワークド例題（Backward Fading） |
| 0.3 ≤ P < 0.8 | **socratic** | Socratic microassessment（ティーチバック・予測プロンプト） |
| P ≥ 0.8 | **confirmation** | 確認質問1問のみ |

### モード別プロンプト戦略

#### direct モード（P < 0.3）—— Worked Example パターン

1. **初回**: 概念説明 + ステップ全公開の例題（「条件・行動・目的」フレームワーク）
2. **2回目**: 最終ステップを空白にした補完問題（「ここまで来たら最後の式は？」）
3. **3回目以降**: ヒントのみ提示 → 独立演習
4. コードブロックで数式と説明を同じ行に記載（Split-Attention 防止）
5. 最低2例題を提示してから独立演習へ（転移学習のため）
6. 「計算量が少ない数値」を例題に使用（認知負荷軽減）

```
例）1例題あたりの構造:
<例題>
条件: 二次方程式 ax²+bx+c=0 の形式が与えられているとき
行動: 判別式 D=b²-4ac を計算する
目的: 実数解の個数を判断するため
</例題>
```

#### socratic モード（0.3 ≤ P < 0.8）—— Socratic microassessment

1. 短い説明 or 解説の後、**確認質問を1問だけ**末尾に追加
2. 質問タイプは以下から選択:
   - **ティーチバック**: 「自分の言葉で〇〇を説明してみて」
   - **予測プロンプト**: 「これを実行するとどうなると思う？」
   - **自己説明**: 「ここで÷mをしたのはなぜだと思う？」（原理の言語化）
   - **反事実**: 「もし最初にyで解いていたらどうなってた？」（IF-NOT）
3. 質問は**文末で必ず終了**。その後にヒントや補足を付け足さない

#### confirmation モード（P ≥ 0.8）—— 確認問題

1. 確認質問**1問のみ**。詳細説明は省く
2. 「できてる感」を維持しつつさらに深める問いかけ
3. 問題が難化しすぎないよう配慮

## ビジネスルール

### 語調・フレーミング（DEC-25）

- BR-05-01: **「伴走者」フレーミング**を全回答で徹底する。「評価」「採点」ではなく「一緒に整理」語句を使う
- BR-05-02: 確認質問の例: 「ここ一緒に確認しておきたいんだけど、〇〇ってどういう意味だったか自分の言葉で教えてもらえるかな？」
- BR-05-03: スキップ要求（「次」「わからない」等）への即時肯定: 「オッケー！じゃあ次に進もう」
- BR-05-04: 過度な敬語を避け、中高生が自然に話せる語調（「〜だよ」「〜してみて」）

### 適応型戦略

- BR-05-05: P(mastery) が不明・未取得のトピックは P=0.2（P(L0) デフォルト）でdirectモードを使用
- BR-05-06: **ミッドコンバーセーション・ドロップバック**: socraticモードで生徒が失敗した場合（signal=incorrect かつ連続2回以上）、即座に directモードへ降格する（次スレッドまで待たない）
- BR-05-07: **トピック切替検出**: スレッド内でトピックが変わったことを検出したら、新トピックのP(mastery)で即座にモード再評価する
- BR-05-08: 試験前モード（exam_mode_until が未来日時）の場合は、常に directモードで要点・スピード優先

### リアクション管理

- BR-05-08a: AI回答をSlackに送信完了したら、FR-01 で付けた 🤔 リアクションを削除する（`reactions.remove` API）。削除失敗はサイレント無視

### 内容・安全

- BR-05-09: スタッフが「スタッフ向けに」「指導方針として」等を含む場合のみ、スタッフ向けの補足を追加してよい
- BR-05-10: レポートに基づく内容と一般知識は区別して表現する（例: 「今月のレポートを見る限り〜」）
- BR-05-11: 他の生徒の情報をプロンプトに含めない。person_idで必ずフィルタする
- BR-05-12: 内部エラー・スタックトレース・APIキー・内部IDをSlack返信に含めてはならない
- BR-05-13: 成績や能力を強く断定しない（「〜が苦手です」→「〜でつまずくことが多そうです」）
- BR-05-14: 画像が読めない場合に推測で断定しない
- BR-05-15: テキストのみの質問は安価・高速なモデルを使用。画像ありは Vision対応モデルを使用

## 受入基準（AC）

### AC-05-01: 正常系テキスト質問への回答
```gherkin
Given 有効なチャンネル紐付けのある生徒チャンネル
And 生徒プロフィール要約が存在する
When 生徒がBotにメンションして数学の問題を質問する
Then Slackの元メッセージのスレッドに回答が返信される
And 回答にはやさしい日本語で途中式または考え方が含まれる
And 回答にAPIキーやシステム内部情報が含まれない
```

### AC-05-02: P<0.3 でワークド例題が提供される
```gherkin
Given 生徒Aの「二次方程式」の p_mastery が 0.20 である
When 生徒Aが二次方程式について質問する
Then 回答に「条件・行動・目的」フレームワークに沿った例題が含まれる
And 確認質問の代わりに補完問題が末尾に置かれる
```

### AC-05-03: P≥0.3 で確認質問が1問挿入される
```gherkin
Given 生徒Aの「因数分解」の p_mastery が 0.55 である
When 生徒Aが因数分解について質問する
Then 回答末尾に確認質問が1問だけ含まれる
And 確認質問の後にヒントや補足が付け足されない
```

### AC-05-04: ミッドコンバーセーション・ドロップバック
```gherkin
Given 生徒AのP値が0.35でSocraticモードが選択されている
When Evaluatorがsignal=incorrectを連続2回検出した
Then 次の返信でdirectモード（例題付き）に即座に切り替わる
And モード変更を次スレッドまで待たない
```

### AC-05-05: 試験前モードでdirectになる
```gherkin
Given 生徒Aの exam_mode_until が未来の日時に設定されている
And 生徒AのP値が0.7（通常ならSocraticモード）
When 生徒Aが質問する
Then directモード（要点重視）で回答される
And 確認質問は含まれない
```

### AC-05-06: スキップ要求の即時受容
```gherkin
Given Botが確認質問を含む返信を送信した
When 生徒が「次」「わからない」と返信した
Then 「オッケー！じゃあ次に進もう」または同趣旨の肯定メッセージが返される
And BKTのP値は変更されない（FR-23の skip=null ルール）
```

### AC-05-07: スタッフ向け補足
```gherkin
Given スタッフが「指導方針として教えて」という表現を含む質問をした
When AI回答が生成される
Then 通常の生徒向け回答に加えてスタッフ向けの補足が含まれる
```

### AC-05-08: レポート情報の出典表示
```gherkin
Given 生徒のレポートに「計算ミスが多い」という記録がある
When 関連する質問への回答が生成される
Then 「今月のレポートを見る限り〜」等、レポート由来であることが分かる表現が使われる
```

### AC-05-09: 他の生徒情報が混入しない
```gherkin
Given 生徒Aと生徒Bがそれぞれ別チャンネルで利用している
When 生徒Aのチャンネルで質問が処理される
Then 生徒Bのプロフィールやレポート情報が回答に含まれない
```
