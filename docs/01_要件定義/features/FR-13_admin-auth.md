# FR-13: 管理画面認証

## 概要

管理画面へのアクセスをメールアドレス+パスワードで認証する。Supabase Authを使用する。認証されていないユーザーは管理画面のすべてのページにアクセスできない。

## アクター

- U-02 スタッフ（ログイン・ログアウト）
- U-03 管理者（ログイン・ログアウト）

## 入力データ

| フィールド | 型 | 必須 | 制約 | 備考 |
|-----------|----|----|------|------|
| email | string | ○ | メール形式 | - |
| password | string | ○ | 8文字以上 | - |

## ビジネスルール

- BR-13-01: 管理画面の全ルートは認証必須。未認証ユーザーはログインページにリダイレクトする
- BR-13-02: ログイン試行の失敗回数制限はSupabase Authのデフォルト設定に従う
- BR-13-03: セッション管理はSupabase Authのセッション（JWT）に委ねる
- BR-13-04: アカウント作成はMVPでは管理者が手動でSupabaseコンソールから行う（登録UIはMVPスコープ外）[仮決定]

## 受入基準（AC）

### AC-13-01: 正常ログイン
```gherkin
Given 有効なメールアドレスとパスワードを入力する
When ログインボタンを押す
Then ダッシュボード（SCR-02）にリダイレクトされる
```

### AC-13-02: 未認証アクセスのリダイレクト
```gherkin
Given 未ログイン状態
When 管理画面の任意のURLに直接アクセスする
Then ログインページ（SCR-01）にリダイレクトされる
```

### AC-13-03: ログアウト
```gherkin
Given ログイン済み状態
When ログアウトボタンを押す
Then セッションが破棄されてログインページにリダイレクトされる
```
