# FR-09: 生徒プロフィール要約

## 概要

各生徒について、AI回答時に常時プロンプトに含める短い要約を管理する。現在の学習状況・苦手分野・得意分野・指導上の注意・説明トーン・直近の重点課題を含む。

## アクター

- U-02 スタッフ（プロフィール要約を作成・編集する）
- U-03 管理者（全操作）
- システム（AI処理時にプロフィール要約を取得する）

## 入力データ

| フィールド | 型 | 必須 | 制約 | 備考 |
|-----------|----|----|------|------|
| person_id | UUID | ○ | - | 1人につき1レコード |
| summary | string | ○ | 1〜2000文字 | 全体的な要約。AI用 |
| learning_style | string | △ | 〜500文字 | 説明トーン・学習スタイル |
| strengths | string | △ | 〜500文字 | 得意分野 |
| weaknesses | string | △ | 〜500文字 | 苦手分野 |
| instruction_notes | string | △ | 〜1000文字 | 指導上の注意点 |

## 出力 / 結果

- 正常系: AI処理のプロンプト先頭部分に挿入されるテキストを返す
- 異常系（レコードなし）: 生徒プロフィールなしで処理を継続（エラーとしない）

## ビジネスルール

- BR-09-01: person_idに対して1レコードのみ存在する（UPSERTで管理）
- BR-09-02: プロフィール要約はAIへの全リクエストに必ず含める（レポートチャンクより優先度高）
- BR-09-03: プロフィール要約の合計トークン数目安は500〜1500 tokens（日本語約400〜1200文字）
- BR-09-04: プロフィールが存在しない場合でも回答は実行する（品質は下がるがエラーにしない）

## 受入基準（AC）

### AC-09-01: プロフィール要約の作成
```gherkin
Given 管理画面でスタッフが生徒Aのプロフィールを入力する
When 保存する
Then student_profilesにUPSERTされる
And person_idで1レコードのみ存在する
```

### AC-09-02: AI処理への組み込み
```gherkin
Given 生徒Aのプロフィール要約が存在する
When 生徒Aのチャンネルで質問を処理する
Then プロフィール要約がAIプロンプトの先頭に含まれる
```

### AC-09-03: プロフィールなしでも回答継続
```gherkin
Given 生徒Bのstudent_profilesレコードが存在しない
When 生徒Bのチャンネルで質問を処理する
Then エラーにならず
And プロフィールなしで（一般的な回答として）AIが回答する
```
