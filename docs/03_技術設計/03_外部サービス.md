## 2. 各サービス詳細

### 2.1 Slack API

**目的**: メッセージ受信・返信・ファイルダウンロード

**必要な設定**:
- Slack App作成 → `api.slack.com/apps`
- Bot Token Scopes:
  - `channels:history` — チャンネル履歴読み取り
  - `channels:read` — チャンネル情報取得
  - `chat:write` — メッセージ投稿
  - `files:read` — ファイルダウンロード
  - `users:read` — ユーザー情報取得（任意）
- Event Subscriptions:
  - Request URL: `https://{your-domain}/api/slack/events`
  - Subscribe to: `message.channels`
- App-Level Token: Socket Modeは使わない（Events API Webhookを使用）

**環境変数**:
```
SLACK_BOT_TOKEN=xoxb-xxxxx
SLACK_SIGNING_SECRET=xxxxx
SLACK_BOT_USER_ID=U_BOTID   # BotのSlack User ID（メンション検出用）
```

**制約**:
- Events API: Slack側のリトライあり（3秒以内にACK必須）
- ファイルURL: `url_private` はBot tokenが必要（有効期限あり）
- レートリミット: `chat.postMessage` は1チャンネルあたり1msg/秒

### 2.2 Supabase

**目的**: PostgreSQL DB + pgvector + Auth + Storage

**プラン**:
- MVP開始: Free（500MB DB・1GB Storage・5万MAU）
- 成長後: Pro $25/月（8GB DB・100GB Storage）

**必要な設定**:
- pgvector拡張: `CREATE EXTENSION IF NOT EXISTS vector;`
- Supabase Auth有効化
- Storage bucket作成: `attachments`（private）
- Service Role Keyの取得

**環境変数**:
```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=xxxxx    # ブラウザ用（公開可）
SUPABASE_SERVICE_ROLE_KEY=xxxxx        # サーバーのみ・絶対に公開しない
```

**制約**:
- Free: 同時接続50、1週間非アクティブでDBが一時停止
- pgvector: IVFFlat インデックスはデータ挿入後に作成すること

### 2.3 Vercel

**目的**: Next.jsホスティング + Serverless Functions + waitUntil

**プラン**:
- Pro $20/月（課金済み）— `waitUntil` 利用・関数タイムアウト最大15分

> **DEC-13**: ジョブWorkerはVercel Cronではなく `waitUntil` で実装。
> Cron は最短1分間隔の制約があり応答遅延が大きい（最悪75秒）。
> `waitUntil` はWebhook受信直後にバックグラウンド実行するため15〜30秒で返信できる。
> jobs テーブルは維持してリトライ・ログ・管理画面での確認を継続する。

**waitUntil の使い方**:
```typescript
// src/app/api/slack/events/route.ts
import { waitUntil } from '@vercel/functions'

export async function POST(request: Request) {
  waitUntil(processJobInBackground(payload))  // バックグラウンドで実行
  return new Response('OK', { status: 200 })  // 即座に返す
}
```

### 2.4 LLM API（Anthropic / OpenAI）[仮決定]

**モデル使い分け案**:
| 用途 | モデル候補 | 理由 |
|------|-----------|------|
| 通常テキスト質問 | claude-haiku-4-5 / gpt-4o-mini | 低コスト・高速 |
| 画像あり質問 | claude-sonnet-4-6 / gpt-4o | Vision性能 |
| 複雑な問題解説 | claude-sonnet-4-6 | 推論品質 |
| レポート要約生成 | claude-haiku-4-5 / gpt-4o-mini | コスト最適 |
| Embedding生成 | text-embedding-3-small (OpenAI) | コスパ |

**コスト試算（月次、生徒50人×週5質問×4週）**:
```
月間質問数: 50人 × 5問/週 × 4週 = 1,000問
通常質問 (700問): input 1,500tok + output 500tok = 2tok/問
画像質問 (300問): input 2,500tok + output 800tok

claude-haiku-4-5 ($0.25/M input, $1.25/M output):
  700問 × 1,500tok = 1,050k input → $0.26
  700問 × 500tok = 350k output → $0.44

claude-sonnet-4-6 ($3/M input, $15/M output):
  300問 × 2,500tok = 750k input → $2.25
  300問 × 800tok = 240k output → $3.60

Embedding (text-embedding-3-small $0.02/M):
  1,000チャンク × 500tok = 500k → $0.01

月額合計（推定）: ~$6〜$10
```

**環境変数**:
```
ANTHROPIC_API_KEY=xxxxx
OPENAI_API_KEY=xxxxx       # Embedding用（Anthropic使う場合も必要）
LLM_MODEL_DEFAULT=claude-haiku-4-5-20251001
LLM_MODEL_VISION=claude-sonnet-4-6
LLM_MODEL_COMPLEX=claude-sonnet-4-6
EMBEDDING_MODEL=text-embedding-3-small
ENABLE_AI=true             # kill switch
```

## 5. リスクと代替策

| サービス | リスク | 代替 |
|---------|--------|------|
| Anthropic API | レートリミット・サービス停止 | OpenAI GPT-4oへの切り替え（抽象化レイヤーで対応） |
| OpenAI Embedding | モデル変更・次元数変更 | Supabase Edge Functions + local embedding |
| Supabase | サービス停止 | Neon + Pinecone（移行コスト高）|
| Vercel Cron (Hobby) | 1job/日制限 | cron-job.org（無料）で代替 |
| Slack API | Bot token失効 | 管理画面でtoken更新フローを整備 |
