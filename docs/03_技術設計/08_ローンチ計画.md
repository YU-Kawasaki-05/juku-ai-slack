# ローンチ計画 — juku-ai-slack-bot

## 1. フェーズ別ローンチ

### Phase 1: 内部α（1名テスト）
- **目的**: 単一の生徒チャンネルで基本的なBotの動作を確認
- **対象**: 開発者自身 or 協力者1名
- **期間**: Sprint 2完了後（約2ヶ月後）
- **スコープ**: FR-01〜05（Slack受信・AI回答）のみ
- **インフラ**: Vercel Preview + Supabase Free

### Phase 2: 限定β（5名テスト）
- **目的**: RAG・画像対応・管理画面の実運用検証
- **対象**: 塾スタッフ1〜2名 + 生徒3〜4名
- **期間**: Sprint 6完了後（約4ヶ月後）
- **スコープ**: FR-01〜17（全P0機能）
- **インフラ**: Vercel Hobby + Supabase Free（本番環境に近い設定）

### Phase 3: 正式ローンチ（全生徒）
- **目的**: 全生徒への本番展開
- **対象**: 塾在籍生徒全員
- **期間**: β後のバグ修正・改善完了後
- **スコープ**: 全P0機能 + 必要に応じP1追加
- **インフラ**: Vercel Pro + Supabase Free（生徒数次第でPro移行）

## 2. ローンチ前チェックリスト

### 2.1 機能チェック
- [ ] 全P0 AC（AC-01〜AC-17）のテストが通過している
- [ ] 生徒チャンネルでのエンドツーエンド動作確認済み
- [ ] 管理画面で生徒・チャンネル・レポートのCRUDが動作確認済み
- [ ] エラー時のSlack返信文言が適切か確認済み
- [ ] RAG検索精度の手動確認済み（レポートあり/なしで違いが出るか）

### 2.2 セキュリティチェック
- [ ] 環境変数に本番値が設定されている
- [ ] Service Role Keyがクライアントに露出していない（Vercel Environment Variablesで SERVER ONLY設定済み）
- [ ] Slack Webhook署名検証が本番URLで動作している
- [ ] 生徒間のデータ越境がないことをテストで確認済み
- [ ] ENABLE_AI=true が設定されている
- [ ] kill_switchテーブルにデータが存在する

### 2.3 コスト対策チェック
- [ ] kill_switchテーブルが機能している（手動でdisabledにして確認）
- [ ] 日次コスト集計Cronが動作している
- [ ] Slackアラートチャンネルが設定されている
- [ ] 月額警告閾値がkill_switch連携で動作している

### 2.4 オペレーション準備
- [ ] 管理者アカウント（Supabase Auth）が作成されている
- [ ] スタッフアカウントが作成されている
- [ ] seedデータ（初期生徒データ・チャンネル紐付け）が投入されている
- [ ] インシデント対応Runbook（06_運用設計.md）を読んでいる
- [ ] Sentryアラート宛先のSlackチャンネルが設定済み

### 2.5 ロールバック手順
- 問題発生時: Vercelダッシュボードから直前のデプロイに1クリックでロールバック
- DBマイグレーション: Supabaseで手動でロールバックSQLを実行（事前に準備）
- Bot動作停止: `ENABLE_AI=false` に変更してVercelを再デプロイ（即座に反映）

## 4. 生徒・スタッフへのオンボーディング

### 生徒向け案内（Slack DM or チャンネル投稿）

```
こんにちは！勉強サポートBot「〇〇先生」を使い始めます。

【使い方】
・このチャンネルで @先生 とメンションして質問してください
・写真（問題用紙）を一緒に送ることもできます
・スレッドで返信するとそのまま会話が続きます

【できないこと】
・テストの答えをそのまま教えることはできません（ヒント形式で返答します）
・学習内容以外の質問への回答

質問があればスタッフまで！
```

### スタッフ向け操作手順書
- 管理画面へのログイン方法
- 新規生徒の追加・チャンネル紐付け手順
- レポートの登録・公開手順
- エラー一覧の確認・対応済みマーク手順

## 5. リスクシナリオと事前対策

| リスク | 確率 | 対策 |
|--------|------|------|
| ローンチ直後のコスト急増 | 中 | kill_switchと日次バッチが自動停止 |
| 生徒が不適切な質問を送信 | 中 | AIシステムプロンプトでスコープ限定 |
| スタッフがチャンネル紐付けを間違える | 高 | UNIQUE制約 + 管理画面で確認UIを整備 |
| Vercel Cronが止まりジョブが積み残す | 低 | アラートで検知 → 手動再実行 |
| Slack Botが他のワークスペースに招待される | 低 | 単一ワークスペース設計（検証は省略）|
