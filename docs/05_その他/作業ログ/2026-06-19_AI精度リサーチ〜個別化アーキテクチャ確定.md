### DEC-20: 簡略化BKT（知識状態追跡）
- updateBKT() をTypeScript純実装（外部ライブラリ不要）
- Zod structured output で {topic_id, correctness, confidence, cognitive_gap} を抽出
- 時間減衰: 週1%（TASA論文準拠）
- 詳細: `docs/00_共通/決定事項ログ_decision-log.md`, `FR-23_bkt-knowledge-tracking.md`

### DEC-21: Open Brain / Duolingo Lilyパターン（エピソード記憶）
- セッション終了後に事実抽出 → 重要度1〜10スコアリング → pgvector保存
- 重要度7以上はブートシーケンスで毎回ロード（ベクトル検索ゼロ）
- Anthropicプロンプトキャッシングでコスト約80%削減
- Supabase RLSでデータ漏洩防止（Mem0方式は不採用）
- 詳細: `FR-24_episodic-memory.md`

### DEC-22: FSRSスペース反復リマインダー（Phase 2）
- `ts-fsrs` npm package（v6対応）
- レビュー単位 = 知識マイクロ概念（誤概念から自動生成）
- LinGoatパターン: 複数due概念を1問に合成
- 詳細: `FR-25_fsrs-reminder.md`

## 採用しなかった技術と理由

| 技術 | 不採用理由 |
|------|-----------|
| Mem0 セルフホスト | Vercel非互換（Docker常駐必要）＋RLSなしでdata漏洩リスク |
| Zep/Graphiti | Neo4j必須で過剰設計 |
| DKT/SAINT | データ不足（数十万件必要） |
| 標準BKT（EM最適化あり） | 大量の事前データ必要、Node.js不向き |
| Felder-Silverman | 科学的根拠弱い（信頼性r≈0.5〜0.6） |
