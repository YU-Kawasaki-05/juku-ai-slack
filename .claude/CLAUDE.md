# juku-ai-slack-bot プロジェクト指示

## 作業ログ

作業ログは `docs/05_その他/作業ログ/` に保存する。

- ファイル単位: 1日1ファイル、または作業テーマが変わったら新ファイル
- ファイル名: `YYYY-MM-DD_テーマ.md`（例: `2026-06-16_要件定義完了.md`）
- 粒度: 作業内容の要点と「詳細はどこを見ればわかるか」のポインタがあればOK。細かすぎなくてよい
- タイミング: 作業の区切りごと（セッション終了前、フェーズ完了時など）に更新する
- **重要**: 新しい作業を始める前に当日のログファイルが存在するか確認し、なければ作成する

## ドキュメント構成

```
docs/
  00_共通/        決定事項ログ・用語集・リスク登録簿
  01_要件定義/    FR-XX.md（機能要件）・_index.yml
  02_外部設計/    DB設計・API仕様・権限設計・テスト戦略
  03_技術設計/    アーキテクチャ・Sprint計画・ガイドライン
  04_整合性チェック/  drift検出レポート
  05_その他/      作業ログ・その他メモ
```

## 重要な設計決定（最新）

- DEC-13: ジョブ処理は `waitUntil + jobs テーブル`（Cron不使用）
- DEC-14: Embedding はレポート保存時に自動再生成
- DEC-15: kill_switch 変化時は Slack #alerts 通知 + 管理画面に状態表示
- 認証: スタッフ・管理者のみ Supabase Auth でログイン。生徒は Slack 経由のみ
- channel_id が信頼の基点。channel_name は表示用のみ
